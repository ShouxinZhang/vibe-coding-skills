# 自举提升（Bootstrapping）

## 定义

自举提升（Bootstrapping）是指 Agent 利用自身执行过程产生的数据、评估结果与反思结论，持续改进策略、提示结构或训练数据，从而在较少外部监督下实现性能提升。

它的业务价值是降低专家介入成本，让系统具备低人力的持续优化能力。

## 自举闭环

1. 执行：在真实任务中产出结果与中间轨迹。
2. 评估：用规则、模型或人工反馈判定质量。
3. 反思：定位失败模式并生成改进建议。
4. 更新：把高价值改进注入提示、策略或数据。
5. 复测：在基准集和线上样本中验证收益。

## 常见实现路径

1. Self-Reflection：让 Agent 生成错误分析与修复计划。
2. Self-Training：将高置信结果转化为伪标签样本。
3. Distillation：把复杂推理蒸馏为可复用流程模板。
4. Policy Refinement：根据反馈迭代决策策略。

## 风险点

1. 错误自增强：低质量样本进入迭代链路会放大偏差。
2. 指标幻觉：只优化代理指标而非真实业务目标。
3. 局部最优：系统早熟收敛，失去探索能力。

## 防失控策略

1. 设置数据准入门槛，过滤低置信度样本。
2. 保留人工抽检通道，定期校准质量基线。
3. 引入探索预算，避免长期只走单一路径。

## 落地建议

1. 先自举流程与提示，不急于全量模型更新。
2. 构建"收益-风险"双指标看板，持续观察。
3. 每次只引入一个变量变更，确保可归因。
