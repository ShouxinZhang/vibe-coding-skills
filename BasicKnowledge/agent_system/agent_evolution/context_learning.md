# 上下文学习（Context Learning）

## 定义

上下文学习（In-Context Learning, ICL）是指 Agent 不更新模型参数，仅通过输入上下文中的指令、示例、约束与历史交互来学习任务模式并产生更优输出。

它的业务价值是缩短冷启动周期：系统无需重新训练即可快速适配新流程、新规范和新数据格式。

## 核心机制

1. 任务描述：明确目标、边界与成功标准。
2. 示例注入：提供少量高质量样例，建立行为模板。
3. 约束表达：显式规定格式、语气、合规规则和禁区。
4. 反例提示：给出错误示例，降低模型偏航概率。

## 设计原则

1. 样例数量少而精，优先覆盖高风险场景。
2. 上下文结构稳定，避免频繁变更字段语义。
3. 指令层级清晰，先目标后约束再样例。
4. 输出协议固定，便于自动校验与回归比较。

## 常见问题

1. 上下文过长导致注意力稀释，核心指令被淹没。
2. 示例质量不一致引发风格漂移与逻辑冲突。
3. 规则冲突未显式解决，模型在多约束下不稳定。

## 落地建议

1. 建立"基础提示模板 + 任务增量补丁"机制。
2. 对关键任务建立上下文 A/B 测试并保留版本。
3. 把高频错误沉淀为反例集持续复用。
