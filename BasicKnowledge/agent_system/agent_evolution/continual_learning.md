# 持续学习（Continual Learning）

## 定义

持续学习（Continual Learning）是指 Agent 在长期运行中持续吸收新任务经验，同时尽量保持既有能力不退化。核心挑战是灾难性遗忘（Catastrophic Forgetting）。

它的业务价值是把一次性交付升级为长期增值：系统上线后能力不是静止，而是可衡量地提升。

## 关键挑战

1. 任务分布变化快，旧经验与新需求存在冲突。
2. 标注反馈稀缺，真实世界中高质量监督信号有限。
3. 回归风险高，新策略可能破坏已有稳定能力。

## 常见策略

1. 经验回放（Replay）：保留代表性样本进行混合训练。
2. 参数约束（Regularization）：限制关键参数剧烈漂移。
3. 模块扩展（Expansion）：为新能力开辟隔离参数空间。
4. 选择性更新：只更新与当前任务强相关的能力模块。

## 评估框架

1. 前向迁移（Forward Transfer）：新知识是否加速未来任务。
2. 反向迁移（Backward Transfer）：学习新任务后旧能力是否下降。
3. 稳定性与可塑性平衡：既不过度保守也不过度漂移。

## 工程建议

1. 建立周期性评估集，固定追踪核心业务指标。
2. 引入回滚阈值，学习收益低于阈值时停止更新。
3. 将反馈链路产品化，确保学习数据可追溯。
