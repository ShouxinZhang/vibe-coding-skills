# 创造 Subagents（Subagent Creation）

## 定义

创造 Subagents 是指主 Agent 根据任务复杂度动态创建具备独立职责、上下文和工具权限的子 Agent，用于并行处理子问题并汇总结果。

它的业务价值是提升复杂任务吞吐：把串行瓶颈变成可并行执行的能力网络。

## 何时创建 Subagents

1. 目标可拆分为多个弱耦合子任务。
2. 子任务需要不同工具链或不同专业策略。
3. 单 Agent 上下文已接近窗口极限。
4. 任务时限要求并行推进。

## 角色模型

1. Orchestrator：负责拆解、分派、收敛与冲突裁决。
2. Specialist：专注单一能力域，追求深度质量。
3. Reviewer：独立评审输出，降低同源错误。

## 设计要点

1. 子 Agent 目标必须可验证，避免模糊任务描述。
2. 权限最小化，只开放完成任务所需工具。
3. 通信协议标准化，统一输入输出格式。
4. 失败可隔离，单点异常不拖垮全局流程。

## 风险与治理

1. 过度拆分导致协调成本超过收益。
2. 子 Agent 结论冲突需要仲裁机制。
3. 追踪链路不清会降低可解释性与审计能力。

## 落地建议

1. 从固定角色模板起步，再逐步引入动态创建。
2. 先在高价值、高复杂度任务中验证收益。
3. 用统一日志记录任务树与决策路径。
