# 意识：Agent 对自身的可观察性与可调节性

## 模块定位

如果说 [../life/introduction.md](../life/introduction.md) 讨论的是系统如何持续存在，[../intelligence/introduction.md](../intelligence/introduction.md) 讨论的是系统如何在约束下做对事，那么 Consciousness 讨论的是另一个更敏感的问题：**系统是否知道自己在做什么，以及这种"知道"应当如何被拆解。**

在工程上，最重要的不是争论 Agent 是否真的拥有主观体验，而是把"意识"拆成几种可以被单独观察、验证和治理的能力。

---

## 为什么需要这个子模块

意识是 Agent 讨论中最容易陷入混乱的词。因为同一句"它有意识"，可能实际上在说三件完全不同的事：

1. 它是否真的有主观体验
2. 它是否能报告自己的状态和理由
3. 它是否拥有稳定的自我模型

如果不把这些维度拆开，讨论很快就会滑向不可验证的哲学争论，反而无助于工程落地。
