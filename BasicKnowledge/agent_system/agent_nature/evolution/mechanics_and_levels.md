# 变异、选择、保留与四层进化

## 一、进化的三个核心机制

进化之所以不是普通改动，而是真正的系统升级，关键在于三个机制的联动。

### 1.1 变异

系统必须能产生差异：不同的提示词策略、不同的工具编排、不同的角色分工、不同的评估流程。没有变异，系统就没有探索空间。

### 1.2 选择

不是所有变异都值得留下。任务成功率、成本、稳定性、用户信任度都会构成选择压力。没有选择，系统只会积累噪声，不会形成方向。

### 1.3 保留

成功的模式必须被记录、复用和传递。否则每一次改进都会在下一轮重新蒸发，系统无法形成复利。

---

## 二、四层进化框架

### 2.1 任务内进化

在单次任务中，Agent 通过反馈和试错快速修正当前策略。这种进化速度快，但通常只影响当前任务。

### 2.2 跨任务进化

系统把一次次任务里的成功经验沉淀到模板、规则、记忆和评估机制中。这一层最直接决定业务复利。

### 2.3 群体进化

多个 Agent 在协作或竞争中逐渐形成更高效的角色分工、通信惯例和组织结构。被选择的不再只是单个策略，而是整体拓扑。

### 2.4 生态进化

当 Agent、用户、工具、平台和制度形成长期耦合，进化对象就从单个 Agent 扩展为整个生态。此时真正起决定作用的，往往是激励与治理设计。

---

## 三、学习和进化的区别

学习强调个体能力在短中期内提升，进化强调哪些变化能跨时间被保留下来，成为系统的新常态。

| 维度 | 学习 | 进化 |
|------|------|------|
| 主体 | 单个 Agent | 个体、群体或系统 |
| 时间尺度 | 短到中期 | 中到长期 |
| 核心机制 | 参数更新、记忆更新、策略调整 | 变异、选择、保留 |
| 结果 | 当前表现提升 | 结构性能力沉淀 |

没有保留机制的学习，很难转化为真正的进化。

---

## 四、进化的真实风险

进化并不天然向善。如果选择信号设计错误，系统会高效地朝错误方向演化。典型风险包括：

- 目标漂移：过度优化代理指标，偏离真实目标
- 路径依赖：早期偶然成功被固化为长期惯例
- 局部最优锁死：探索空间过早关闭
- 生态污染：短期收益破坏长期结构健康

因此，进化必须和 Gate、回滚、审计一起设计。

---

## 五、工程启示

真正有价值的进化，不是让系统越来越复杂，而是让它在面对新任务时更快收敛、更少犯错、更容易复制成功经验。从业务角度看，这决定了 Agent 是一次性项目，还是可持续复利的生产资产。
