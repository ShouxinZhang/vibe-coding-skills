# 人格系统：基于记忆切片的行为策略

## 核心定位

人格（Persona）不是情绪模拟，而是可配置的决策风格。它由记忆切片驱动，决定 Agent 在速度、风险、解释深度之间如何取舍。

对业务的意义在于：同一套能力底座，可以按场景切换成不同执行风格，而不是训练多套完全独立系统。

## 三种基础人格

| 人格 | 特征 | 适用场景 |
|------|------|---------|
| 审慎型 | 重验证、低风险偏好 | 生产变更、权限类任务 |
| 探索型 | 高迭代、快试错 | 研发探索、方案发散 |
| 执行型 | 平衡风险与效率 | 日常交付、标准流程 |

## 记忆切片如何塑造人格

人格建议基于以下记忆信号生成：

1. 历史成功率与失败类型分布。
2. 重试与回退的比例。
3. 平均规划时长与执行时长。
4. 高风险任务中的合规通过率。

这些信号决定了默认参数，例如检查点频率、重试上限、超时阈值。

## 收益与风险

### 收益

1. 任务匹配更精准：高风险场景默认更保守。
2. 组织效率更高：探索任务不再被过度流程束缚。
3. 决策更可解释：人格参数可审计、可对比。

### 风险

1. 刻板化风险：人格标签固化后，可能压制新策略。
2. 漂移风险：长期自适应可能让人格偏离初衷。
3. 合规风险：不当人格被用于高约束任务。

## 控制机制

1. 为高风险任务设置强制人格白名单。
2. 定期复盘人格参数漂移并设置上下限。
3. 记录每次人格选择依据，支持审计回溯。

## 最小可落地版本（MVP）

1. 固定三种人格模板（审慎/探索/执行）。
2. 用规则映射任务到默认人格。
3. 在执行上下文注入少量关键参数：重试上限、检查点频率、超时阈值。
4. 每周输出一次人格效果报告（成功率、耗时、回退率）。

该 MVP 的价值在于先建立"风格可控"能力，再逐步引入更复杂的自适应机制。
