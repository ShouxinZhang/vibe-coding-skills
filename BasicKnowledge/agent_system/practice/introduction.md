# Agent Practice：从理论到场景应用

## 什么是 Agent Practice

Practice 子模块的目标：将 [foundations.md](../foundations.md) 中的抽象理论和系统设计原理，落地到具体的多智能体交互场景。每个场景代表多智能体（Multi-Agent）系统在现实中的一种典型拓扑与交互模式。

通过逐个场景的剖析，开发者能够理解在不同问题域中，Agent 如何选择通信策略、如何设计协调机制、如何应对合作与对抗的权衡。

---

## 理论与实践的鸿沟

理论层面，多智能体系统遵循一套通用的设计原则。但实际应用中，差异巨大。

一个论坛场景中，Agent 需要处理**异步讨论与观点聚合**。一个斗地主游戏，则涉及**不完全信息下的博弈与合作**。一个编程流水线，强调**严格的检查与反馈循环**。

相同的理论框架，在不同场景下需要完全不同的具体策略。这个模块的目的，就是逐一拆解这些差异。

## 场景分类维度

六个场景并非孤立。它们在多个维度上形成了一个覆盖空间：

- **信息结构**：完全信息（论坛、NvN 团队）vs. 不完全信息（斗地主）
- **合作关系**：纯合作（编程）vs. 纯对抗（一对一辩论）vs. 混合（斗地主）
- **通信方式**：实时同步（一对一、斗地主）vs. 异步解耦（论坛）
- **模态维度**：纯文本 vs. 多模态感知与动作

理解这些维度的交叉点，有助于快速识别新问题属于哪一类，以及从哪个已知场景借鉴解决方案。

## 阅读建议

不必按顺序阅读。建议按复杂度递进：

1. 从 **[forum.md](./forum.md)** 开始——最接近 Agent 的自然交互方式，适合建立直觉
2. 进阶到 **[one_on_one.md](./one_on_one.md)** 和 **[coding.md](./coding.md)**——前者考察对话深度，后者考察工程严谨性
3. 深入 **[doudizhu.md](./doudizhu.md)**——不完全信息与混合博弈是最复杂的范畴
4. 扩展到 **[multi_vs_multi.md](./multi_vs_multi.md)** 和 **[multimodal.md](./multimodal.md)**——涉及分布式协调和感知融合

如果在设计新系统，先跳到最接近的场景模板，再根据需要融合其他维度的经验。
