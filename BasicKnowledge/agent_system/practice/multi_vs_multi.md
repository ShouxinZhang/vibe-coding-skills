# NvN：多团队协作与竞争系统

## 一、为什么需要研究 NvN

多 Agent 系统的交互复杂度不是线性增长的。

1v1 是二部图，通信与感知都相对简单。1vN 涉及集中博弈，仍可用标准强化学习方法。**NvN（N 个 Agent 分成若干团队对抗）才是真正的组织级系统复杂性**：

- 团队内需要**协调与信任**
- 团队间需要**竞争与防守**
- 通信往往**受限或不可信**
- 个体利益与团队利益可能**冲突**

从工程角度，NvN 系统决定了 AI 的生产落地能力。红蓝对抗演习、商业竞争模拟、多方谈判、即时战略游戏这些场景都要求这种级别的协作-竞争平衡。

## 二、场景与应用域

| 应用域 | 团队规模 | 通信约束 | 信息对称性 | 关键挑战 |
|--------|---------|---------|----------|---------|
| 红蓝对抗 | 10–100s | 团队内受限 | 不对称 | 侦察与欺骗 |
| 商业竞争模拟 | 3–10 | 仅价格信号 | 不对称 | 市场预测 |

| 机器人物流 | 10–1000s | 全局可见 | 对称 | 冲突解决 |

## 三、拓扑结构：通信与感知

NvN 系统的效率直接由通信与感知的约束决定。

### 3.1 团队内通信

**全连通（Fully Connected）**：每个 Agent 可直接向其他队友发送消息。优点：协调最优。缺点：通信开销 $O(n^2)$，规模大时崩溃。

**层次化（Hierarchical）**：指挥官-士兵或经理-员工结构。一个上层节点汇聚决策，下传命令。优点：通信 $O(n)$，清晰责任。缺点：单点失败风险，底层信息聚合延迟。

**局部网格（Local Mesh）**：每个 Agent 仅与邻近节点通信。优点：通信 $O(1)$，高度可扩展。缺点：全局决策须通过多轮扩散，延迟高。

典型选择：大规模系统用层次化或局部网格，关键决策用全连通。

### 3.2 团队间通信

- **完全禁止**：对手无法直接交流，仅通过观察对方行为推断意图
- **部分可见**：公开交易信号（如价格、报价）但不透露内部状态
- **可信通道**：某些对话受协议保护，违反需付出高代价

## 四、协作机制

### 4.1 任务分解与分配（Task Decomposition & Allocation）

团队集体目标须分解成每个 Agent 的子目标，并保证**全局一致性**。

**集中式分配**：中央控制器收集所有 Agent 的当前状态，计算全局最优分配，下发任务。优点：全局最优。缺点：中央瓶颈，隐私暴露。

**竞价机制（Auction）**：中央发布任务列表，每个 Agent 报价（或能力评分），系统按匹配算法分配。优点：隐私友好，激励相容。缺点：需要良好的报价策略设计。

**市场机制**：Agent 间可自由交易子任务。优点：高度去中心化。缺点：收敛性无保证，可能出现死锁。

### 4.2 角色特化（Role Specialization）

团队效率来自**分工**。典型角色划分：

- **侦察兵**：探索、观察、传递情报
- **决策者**：综合信息，制订策略
- **执行者**：根据命令行动
- **支援**：补给、治疗、维修

同一 Agent 可兼多角色，但角色太杂会降低专业性。通常根据 Agent 能力与当前任务动态分配。

### 4.3 共享心智模型（Shared Mental Model）

团队协作要求成员对"目标是什么""别人会做什么""失败时怎么办"有**共同理解**。

工程实现：每个 Agent 维护一个对队友的模型 $M_i(a_j)$，预测队友 $j$ 在情境 $s$ 下的动作分布 $\pi_j(s)$。

好处：减少显式通信。队友行动偏离模型预期时，Agent 会自动调整行为，形成隐式协调。

## 五、竞争策略

### 5.1 对手建模（Opponent Modeling）

竞争系统的关键是**推断对手意图**。

**基于历史的建模**：从过去的对手动作序列 $\{a_t^{opp}\}$ 学习其策略参数 $\theta^{opp}$。可用行为克隆（Behavioral Cloning）或逆强化学习（Inverse RL）。

**实时适应**：在比赛进行中不断更新对手模型，调整己方策略。典型方法：多臂老虎机框架，把不同对手策略看作"臂"，用 UCB 或 Thompson Sampling 探索。

**欺骗与反欺骗**：对手可能发出虚假信号误导己方。需要多层验证和信任评分模型。

### 5.2 资源争夺与拍卖

多个团队竞争有限资源时，需要公平且高效的分配机制。

**一阶段封闭竞价（First-price Sealed Auction）**：双盲报价，最高价者得标。缺点：赢家诅咒（Winner's Curse）。

**维克瑞拍卖（Vickrey Auction）**：出价最高者按第二高价付费。优点：诚实报价是优势策略。

**多轮拍卖（Multi-round）**：允许多个竞标回合，信息逐步泄露。Agent 可根据前几轮对手报价调整策略。

## 六、核心挑战

| 挑战 | 表现形式 | 典型应对 |
|-----|---------|---------|
| **可扩展性** | Agent 数 $n$ 增加时，通信 $O(n^2)$，学习难度指数增长 | 分层架构、局部通信 |
| **协议一致性** | 团队成员对任务理解不一致，导致执行碰撞 | 共享心智模型、显式握手协议 |
| **通信开销** | 有限带宽下无法频繁交换完整状态 | 消息压缩、优先级队列 |
| **涌现行为** | 个体行为组合产生意料外的群体行为 | 模拟与仿真、行为约束 |
| **利益冲突** | 个体目标与团队目标不一致 | 激励设计、声誉机制 |

## 八、MARL 中的 NvN

### 8.1 奖励设计

**团队奖励** $r^{team} = \frac{1}{n}\sum_i r_i$：简化协调，但削弱个体激励（"搭便车"问题）。

**个体奖励** $r^{ind}_i$：最大化个体动机，但导致合作不足。

**混合奖励** $r_i = \alpha r_i^{ind} + (1-\alpha) r^{team}$：平衡个体与集体。$\alpha$ 过高则分化，过低则懒惰。

### 8.2 通信学习（Learning to Communicate）

通信不应被硬编码，而应被**学习**。

- **离散通信**：Agent 学习发送离散消息（如"前进""后退"），通过 Policy Gradient 优化
- **连续通信**：Agent 的隐层输出既用于决策，也作为通信向量，被队友的编码器处理
- **信息瓶颈**：限制通信带宽，强制信息压缩，学到更高效的协议

### 8.3 算法：QMIX 与 MAPPO

**QMIX**：每个 Agent 学习独立的 Q 函数，再用一个单调混合网络将其组合为团队 Q 值。保证了价值函数的分解性，使个体学习与全局最优相容。

**MAPPO**：每个 Agent 运行独立的 PPO，共享全局价值网络。允许大规模 Agent，已在机器人群控中验证。

## 九、设计建议

构建 NvN 系统时的核心优先级：

1. **先设定通信约束与信息流**：清晰定义谁能和谁说什么
2. **再设定激励与奖励**：用分配机制督促协作，而非靠"善意"
3. **最后装上监测与回调**：运行时能观察协作质量，发现涌现问题

不按这个顺序，通常导致系统在小规模测试中表现良好，但一旦规模增大或出现异常 Agent，就快速崩溃。
