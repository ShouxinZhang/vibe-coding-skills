# Agent Workflow：无人值守的自动化流水线

## 什么是 Agent Workflow

Agent Workflow，是一套完全由 AI agent 驱动、无需人类实时介入的任务执行体系。

它不是"加了 AI 的脚本"，也不是"辅助人类决策的工具"。它是一条流水线——接收任务，理解上下文，分解步骤，调用工具，验证结果，输出产物，记录日志——整个过程在无人监督的状态下完成。

人类在这套体系里的角色，是**设计者**，不是**操作者**。

---

## 为什么需要它

传统的人机协作模式是：人提需求 → AI 给建议 → 人判断 → AI 执行 → 人验收。

这套模式的瓶颈在人。人的注意力是稀缺的，人的响应速度是有限的，人在重复决策中会疲劳。

Agent Workflow 把这条链路压缩为：**人提需求 → Agent 端到端完成**。

中间的判断、执行、验证，由 agent 承担。人只在两端出现：任务的起点，以及最终结果的审阅。

---

## 这个模块讲什么

本模块记录构建可靠 Agent Workflow 的核心知识，包括：

- **Workspace Structure（工作区结构）**：agent 如何感知和维护工作区，使其在无人值守下不产生混乱
- **Gate（门禁）**：agent 如何在任务关键节点自我验证，阻断错误传播
- （后续扩展）日志与审计、任务分解策略、多 agent 协作……

---

## 核心前提

Agent Workflow 不是银弹。它的可靠性取决于 Meta 的质量。

一个没有 Workspace Structure 约束的 agent，会在第三次任务后把目录搞乱。一个没有门禁的 agent，会把一个小错误悄悄放大成系统级故障。

**自动化的稳定性，来自规则的严密性**。这个模块的每一个章节，都是在为"让 agent 可以安全地独自工作"这个目标铺砖。
