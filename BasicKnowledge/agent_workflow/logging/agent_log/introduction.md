# Agent Log：工作记忆的持久化

## 一句话定位

Agent Log 是 agent 的外部记忆——把上下文窗口中会消失的运行时状态，固化到磁盘上，供下一轮任务恢复。

## 为什么 Agent 需要自己的日志系统

LLM agent 有一个根本缺陷：**无状态**。

每次对话开始，上下文窗口是空的。上一轮任务中 agent 知道的所有事——改了哪些文件、踩了什么坑、做了哪些关键取舍——全部归零。如果没有外部记忆，agent 会重复犯同一个错误，会修改已经被修复的文件，会对工作区的现状做出错误假设。

Agent Log 解决的问题：**让无状态的 agent 在有状态的工作流中保持连续性。**

没有它，全自动化流水线只是一个反复从零开始的循环。有了它，每一轮任务都站在上一轮的肩膀上。

## 与 Human Log 的根本区别

| | Agent Log | Human Log |
|---|---|---|
| **读者** | 机器（下一个 agent） | 人类审阅者 |
| **格式** | 结构化 JSON / SQLite，字段精确 | Markdown 叙事，自然语言为主 |
| **设计目标** | 可查询、可解析、可直接用于决策 | 可阅读、可理解、可追溯 |
| **写入节奏** | 实时，事件驱动 | 任务结束后，关账一次性写入 |
| **容错方向** | 宁可冗余也不能丢字段 | 宁可精简也不能啰嗦 |

一句话总结区别：Human Log 是给人讲故事，Agent Log 是给机器喂数据。
