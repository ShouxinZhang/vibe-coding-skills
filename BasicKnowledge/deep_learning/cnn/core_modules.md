# 核心模块：让 CNN 既看得见，也训得动

> 学 CNN 时，最容易出现一种错觉：好像它只是“卷积层堆很多层”。其实真正让 CNN 变成成熟工程体系的，不只是卷积本身，而是一整套配套模块。它们分别解决不同问题：有的负责提特征，有的负责控制尺寸，有的负责稳定训练，有的负责把网络做深，有的负责把分类头做得更轻。

---

## 一、先把整体图景建立起来

一张图像进入 CNN 之后，通常会经历这样一条路径：

1. 用卷积层提取局部模式。
2. 用 padding 控制边界信息和尺寸变化。
3. 用 stride 或池化逐步缩小空间分辨率。
4. 用 BN 让训练更稳定、更容易收敛。
5. 用残差连接让网络能堆得更深。
6. 在最后用全局平均池化把空间信息压成类别判断需要的摘要。

所以这些模块不是彼此孤立的“知识点”，而是一条完整生产线上的不同工位。

---

## 二、卷积层：负责发现局部模式

### 2.1 它解决了什么问题

卷积层（Convolution Layer）解决的第一个问题，是如何用更少参数去理解图像。

如果直接用全连接层处理图像，参数会大得很夸张，而且模型并不知道哪些像素彼此相邻。卷积层的思路更自然：不一上来就看全图，而是先看一个小窗口，比如 $3 \times 3$ 或 $5 \times 5$，问一句话：

这里有没有某种我关心的局部模式？

这个模式可能是边缘、纹理、角点，也可能是更高层的局部结构。

### 2.2 它为什么有效

卷积层之所以适合图像，本质上抓住了两个事实：

1. 图像中的规律通常先发生在局部。
2. 同一种局部模式会在不同位置重复出现。

所以 CNN 不需要为左上角的边缘学一套参数，也不需要为右下角的边缘再学一套参数。它让同一个卷积核在整张图上滑动，这就是参数共享。

### 2.3 直觉上它在做什么

可以把一个卷积核理解成一个小探测器。它在整张图上反复扫，扫到“像自己想找的模式”的地方，就给出高响应；扫到不相关区域，就给出低响应。

多个卷积核一起工作时，就像同时派出多组侦察兵：

- 有的负责找水平边缘
- 有的负责找垂直边缘
- 有的负责找纹理
- 有的负责找局部形状

这些响应图叠在一起，就形成新的特征图（Feature Map）。

### 2.4 卷积层的代价和限制

卷积层很强，但它也有天然偏好：

- 它更擅长局部结构明显的任务。
- 它不是天生擅长超长距离依赖。
- 如果一味堆卷积而不控制尺寸和训练稳定性，网络很快就会变得又大又难训。

这就是后面那些模块要出场的原因。

## 三、Padding：解决边界信息丢失和尺寸缩水

### 3.1 为什么需要 padding

如果卷积核直接在原图上滑动，不做任何补边，那么它在边界附近能看到的信息会更少，而且每做一次卷积，特征图尺寸通常都会变小。

这会带来两个现实问题：

1. 图像边缘的信息更容易被忽略。
2. 网络一旦堆深，尺寸会缩得很快，后面几层几乎没东西可看。

Padding 的作用，就是在输入周围补上一圈像素，最常见的是补零。

如果输入大小为 $H \times W$，卷积核大小为 $k$，padding 为 $p$，步长为 $s$，那么输出尺寸通常写成：

$$
H_{out} = \left\lfloor \frac{H + 2p - k}{s} \right\rfloor + 1,
\qquad
W_{out} = \left\lfloor \frac{W + 2p - k}{s} \right\rfloor + 1
$$

这条公式的工程意义很直接：padding 不只是“补边缘”，也是尺寸规划工具。

### 3.2 它解决的核心矛盾

Padding 本质上是在平衡两件事：

- 既想做局部扫描
- 又不想太快丢掉边界和空间尺寸

你可以把它理解成给卷积层多留一点操作空间，让边缘位置也能被公平地处理。

### 3.3 常见两种直觉

Valid convolution：不补边，卷积核只在完全放得下的位置滑动。特点是简单直接，但尺寸会缩小。

Same convolution：通过 padding，让输出尺寸尽量和输入接近。特点是更适合深层堆叠，因为空间尺寸不会过早塌掉。

在最常见的奇数卷积核、且 $s=1$ 的情形下，若希望输出尺寸基本不变，通常取：

$$p = \frac{k-1}{2}$$

### 3.4 工程直觉

Padding 不是“数学小修饰”，而是很实际的结构控制器：

- 浅层如果过早丢边界，模型会少看很多细节。
- 深层如果尺寸收缩太快，后面层的表达空间会不够。
- 做检测、分割这类任务时，边缘信息尤其重要。

## 四、Stride：让卷积一边看，一边下采样

### 4.1 它解决了什么问题

Stride 决定卷积核每次滑动几格。默认步长是 1，如果步长变成 2，卷积核就不是逐格细扫，而是“隔一个位置再看”。

它解决的问题是：如何在提取特征的同时，顺手把尺寸缩小。

从公式上看，stride 的作用已经包含在输出尺寸公式里：当 $s$ 从 1 变成 2 时，$H_{out}$ 和 $W_{out}$ 会近似减半。这也是为什么 stride 常被拿来当“可学习下采样”。

### 4.2 为什么这很有用

随着网络加深，特征图如果一直保持很大，计算量会越来越高。可另一方面，越往后走，模型通常越关心“有没有这个模式”，而不是“这个模式精确落在第几行第几列”。

所以在合适的阶段缩小分辨率，是合理的。

步长卷积的好处在于：

1. 它不是纯粹丢信息，而是一边卷积一边筛选。
2. 它比单纯的尺寸压缩更“带任务意识”，因为卷积核本身是可学习的。

### 4.3 它的代价

Stride 变大之后，分辨率会下降得更快，这意味着：

- 计算更省了
- 但细粒度空间信息也更容易丢失

这在分类任务里通常可以接受，但在检测、关键点定位、语义分割这类任务里要更谨慎，因为这些任务更依赖精确位置。

### 4.4 一个简单判断原则

如果任务更重视“是什么”，步长下采样通常更容易接受。

如果任务更重视“在哪儿”，就要对 stride 保守一点。

## 五、池化：用固定规则做压缩和鲁棒化

### 5.1 池化到底在解决什么

池化是早期 CNN 里非常重要的模块。它的目标很直接：

1. 降低特征图尺寸
2. 保留更重要的信息
3. 提高对小幅平移或局部扰动的稳定性

你可以把池化理解成一种“摘要操作”。它不再精确记录每个局部细节，而是提炼一个区域里的代表信息。

### 5.2 最大池化为什么常见

最大池化会在一个小窗口里取最大值。它背后的直觉很朴素：

我不一定关心模式在窗口里的精确位置，但我关心它有没有明显出现。

这和很多视觉任务很契合。比如判断一张图里有没有猫耳朵，局部特征只要强烈出现过，位置稍微偏一点通常不是大问题。

最大池化常写成：

$$y_{i,j} = \max_{(u,v) \in \mathcal{R}_{i,j}} x_{u,v}$$

### 5.3 平均池化在干什么

平均池化则是取窗口内平均值，更像是在问：

这个区域整体上有多强的激活趋势？

它不像最大池化那样强调“最强证据”，而是更强调整体统计。

平均池化对应的写法则是：

$$y_{i,j} = \frac{1}{|\mathcal{R}_{i,j}|} \sum_{(u,v) \in \mathcal{R}_{i,j}} x_{u,v}$$

### 5.4 池化的优点和局限

优点：

- 结构简单
- 不引入额外参数
- 计算代价低
- 对小扰动更稳

局限：

- 它是固定规则，不是可学习的
- 一旦池化太多，细节会损失得比较狠
- 现代网络中，很多位置会用 stride convolution 部分替代池化

所以池化不是过时，而是它的使用方式变得更谨慎了。

## 六、Batch Normalization：让训练过程不再那么脆弱

### 6.1 它解决了什么问题

CNN 真正难的地方，往往不是“写出网络结构”，而是“把它稳定训起来”。网络一深，常见问题就会出现：

- 某些层的输入分布不断变化
- 学习率不好调
- 梯度传播不稳定
- 收敛慢，训练很敏感

BN 就是为了解决这种训练不稳定问题。

### 6.2 直觉上它在做什么

BN 会对一个 batch 中的激活做标准化，让它们回到一个更可控的分布范围，然后再乘上可学习的缩放参数、加上可学习的平移参数。

不用把它想得太数学。更直观的理解是：

每一层都别让自己的输入尺度飘得太离谱。

这件事的价值很大，因为深层网络最怕前面层输出忽大忽小，导致后面层训练节奏被打乱。

如果写成标准公式，BN 在一个 batch 上会做：

$$
\hat{x}_i = \frac{x_i - \mu_{\mathcal{B}}}{\sqrt{\sigma_{\mathcal{B}}^2 + \epsilon}}
$$

$$
y_i = \gamma \hat{x}_i + \beta
$$

其中 $\mu_{\mathcal{B}}$ 和 $\sigma_{\mathcal{B}}^2$ 是当前 batch 的均值和方差，$\gamma, \beta$ 是可学习参数。直觉上就是：先把尺度拉回稳定区间，再允许网络自己学“最终该放大多少、平移多少”。

### 6.3 BN 带来的现实收益

BN 在 CNN 时代之所以影响极大，是因为它通常会带来这些效果：

1. 训练更稳定
2. 可以用更大的学习率
3. 收敛更快
4. 对初始化没那么敏感
5. 某种程度上还有一点正则化效果

它让很多“理论上能训练”的网络，第一次在工程上“真的容易训练”。

### 6.4 BN 的使用位置

在经典 CNN 里，经常会看到这样的顺序：

`卷积 -> BN -> 激活函数`

这个顺序的核心目的，是先把卷积输出整理到更稳定的尺度，再交给非线性函数处理。

### 6.5 BN 不是万能药

BN 很强，但它也有边界：

- 依赖 batch 统计量，小 batch 时效果可能不稳定
- 训练和推理阶段使用的统计方式不同
- 某些场景下会被 LayerNorm、GroupNorm 等替代

不过在 CNN 的历史里，BN 仍然是把“深层模型可训练性”大幅提高的关键模块。

## 七、残差连接：解决“网络更深却更难学”的问题

### 7.1 更深为什么不一定更好

直觉上，网络更深，表达能力应该更强。但早期研究发现一个反常现象：

层数加深以后，模型不只是更难优化，甚至训练误差都可能变差。

这不是简单的过拟合问题，而是优化变得困难了。理论上更深网络至少可以学成“不比浅层差”，但现实里它就是学不到。

### 7.2 残差连接的核心思想

残差连接把输入直接绕过若干层，再和这些层的输出相加：

$$y = F(x) + x$$

这条公式很短，但意义非常大。它等于在告诉网络：

你不必每次都从头学习一个完整变换，你只需要学习“在原有基础上改多少”。

这会让优化简单很多。

### 7.3 为什么“学增量”更容易

很多时候，理想映射并不是彻底推翻输入，而是在输入基础上做微调。残差连接把这个想法结构化了：

- 如果中间几层暂时学不好，至少还有一条捷径把原信息传过去。
- 如果需要细修，网络只学那个差值就行。
- 梯度反向传播时，也多了一条更顺畅的通路。

所以残差连接同时改善了前向信息流和反向梯度流。

### 7.4 它带来的工程意义

残差连接是 ResNet 成功的核心。它让上百层网络不再只是论文里的设想，而变成可以大规模训练、广泛部署的现实架构。

从工程角度看，它解决的是一句话：

不是让网络更深这件事本身更酷，而是让“把深度变成有效能力”这件事终于可行。

## 八、Global Average Pooling：把最后的分类头做轻

### 8.1 它出现之前有什么问题

早期 CNN 往往在最后接上很大的全连接层，把整张高维特征图展平后做分类。这种方式能用，但问题也明显：

- 参数量大
- 容易过拟合
- 对空间位置过度敏感
- 分类头往往比前面特征提取部分还“重”

GAP 就是为了把这一段做得更轻、更自然。

### 8.2 它在做什么

GAP 的做法非常直接：对每个通道在空间维度上取平均，把一整张特征图压成一个数。

如果最后有 512 个通道，那么做完 GAP 后就得到一个 512 维向量。这个向量可以理解成：

每种高级特征在整张图上总体出现得有多强。

如果第 $c$ 个通道的特征图大小为 $H \times W$，那么 GAP 会把它压成：

$$
g_c = \frac{1}{HW} \sum_{i=1}^{H} \sum_{j=1}^{W} x_{c,i,j}
$$

### 8.3 为什么这很合理

在分类任务的最后阶段，模型往往已经不需要太精确的位置细节，而更关心“这类模式整体有没有明显出现”。

比如判断一张图是不是狗，最后一层可能已经学出了很多“耳朵”“鼻子”“毛发纹理”这类高层特征。GAP 做的，就是把这些特征的全图响应浓缩成最终判断依据。

### 8.4 它带来的好处

1. 大幅减少参数量
2. 降低过拟合风险
3. 让结构更简洁
4. 让最后分类决策更接近“按通道汇总证据”

这也是为什么现代 CNN 里，GAP 常常替代大而重的全连接分类头。

## 九、空洞卷积：在不明显增参的前提下扩大视野

### 9.1 它为什么会出现

普通卷积如果想看更大范围，常见办法有两种：

1. 用更大的卷积核
2. 多做几次下采样再往后堆

但这两种办法都有代价。前者会增大参数和计算，后者会损失空间分辨率。

空洞卷积提供了第三条路：在卷积核元素之间“插空”，让同样大小的核看到更大的感受野。

如果卷积核大小为 $k$，空洞率为 $d$，那么它的有效卷积核大小可以写成：

$$k_{eff} = k + (k - 1)(d - 1)$$

所以一个 $3 \times 3$、空洞率为 2 的卷积，感受野效果上会更接近 $5 \times 5$，但参数量并没有真的变成 $25$ 个权重。

### 9.2 它解决的核心问题

它特别适合这种场景：

我既想看更大范围的上下文，又不想太快丢掉分辨率。

这在语义分割、密集预测、一些检测任务里很有价值，因为这些任务既要理解整体，又要保留位置细节。

### 9.3 怎么理解它

你可以把普通 $3 \times 3$ 卷积想成紧密贴在一起的九个采样点。空洞卷积则像是把这九个点拉开间距，让它们覆盖更大的范围。

所以它不是“更多参数”，而是“同样数量的采样点，看更大一圈区域”。

### 9.4 它的代价

空洞卷积也不是白拿好处：

- 采样会更稀疏
- 某些设置下可能出现网格效应
- 如果堆叠方式不合理，局部连续信息反而会变差

所以它通常不是 CNN 的默认基础件，而是针对特定任务需求引入的增强件。

## 十、把这些模块串起来看，CNN 就清楚了

如果把这些模块放在同一条逻辑线上，CNN 的设计思路会非常清楚：

| 模块 | 主要解决的问题 |
|------|----------------|
| 卷积层 | 用更少参数提取局部模式 |
| Padding | 保住边界信息，控制尺寸缩水 |
| Stride | 在提特征的同时做下采样 |
| 池化 | 用固定规则做压缩和增强鲁棒性 |
| BN | 让训练更稳定、更容易收敛 |
| 残差连接 | 让更深网络变得可优化 |
| GAP | 用更轻的方式完成最后分类 |
| 空洞卷积 | 在保留分辨率的同时扩大感受野 |

你会发现，这些设计几乎都不是为了“让数学更漂亮”，而是为了回答非常现实的工程问题：

- 参数太多怎么办？
- 尺寸怎么控制？
- 网络太深怎么训？
- 最后的分类头怎么做得更轻？
- 想看更大范围但又不想丢细节怎么办？

CNN 之所以成为经典，不是因为某一个模块特别神奇，而是因为它把这些问题逐步拆开，并用一套彼此配合的模块把它们解决了。

---

## 十一、小结

如果只记一句话，可以记这个：

卷积负责看，padding 和 stride 负责控制空间，池化负责压缩，BN 负责稳，残差负责把网络做深，GAP 负责把最后输出做轻。

从学习顺序上，建议先真正理解卷积、padding、stride 和池化，因为它们决定了 CNN 如何处理图像；再理解 BN 和残差，因为它们决定了 CNN 为什么能在工程上训得起来；最后再看 GAP 和空洞卷积，就会更容易把“分类任务”和“密集预测任务”的差别串起来。

看懂这些模块后，再去读架构演化，会更容易明白为什么 LeNet、AlexNet、VGG、ResNet、EfficientNet 看起来不同，但本质上都在围绕同一批核心问题持续改进。
