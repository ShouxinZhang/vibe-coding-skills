# 循环神经网络与序列建模

> 本文是 `BasicKnowledge/deep_learning/rnn_lstm/` 子模块的入口。它不急着上公式，而是先讲清楚：为什么深度学习里一定会出现“序列建模”这条分支，RNN、LSTM、GRU、Seq2Seq、Attention 又是怎样一步步长出来的。

## 一、为什么要把序列建模单独拿出来讲？

在 [neural_networks.md](../neural_networks.md) 里，神经网络主要处理的是“当前输入是什么，就直接给出当前输出”。在 [cnn/introduction.md](../cnn/introduction.md) 里，卷积神经网络进一步解决了图像里的空间结构问题。

但文本、语音、时间序列和视频帧流不一样。它们真正难的地方，不是某一个位置长什么样，而是：

- 前后顺序不能乱
- 长度经常变化
- 很多关键信息要隔很多步才能连起来

一句“他因为昨天加班太晚，所以今天没有来开会”，如果只看最后几个词，很难知道“没有来”的主语是谁；如果没有前文，也不容易理解原因。序列建模要解决的，本质就是：**模型怎么带着上下文一路往前走**。

所以，RNN 这一支不是普通神经网络的边角料，而是深度学习第一次系统回答“模型如何拥有记忆”的尝试。

## 三、技术演化时间线

### 1. RNN：先让模型“有记忆”

最早的核心想法很直接：既然序列是一段一段来的，那模型每看一步，就把当前内容和“上一刻的内部状态”合起来，形成新的内部状态。这个内部状态，就是隐状态（Hidden State）。

这是一个很重要的变化。前馈网络只会“当下判断”，RNN 开始尝试“带着过去判断现在”。

但很快大家发现，原始 RNN 理论上能记很远，实践上却常常记不远。原因不是概念错了，而是训练时梯度沿时间链传回去会越来越弱，长依赖学不动。

### 2. LSTM / GRU：再让模型“知道该记什么、忘什么”

RNN 最大的问题，不是它没有记忆，而是它的记忆太粗暴。新的输入一来，旧状态就被整体改写。于是工程上就出现了一个更聪明的问题：

**能不能不要每一步都整本重写，而是让模型自己决定哪些东西保留，哪些东西丢掉，哪些东西现在要写进去？**

LSTM 的答案是：加一套门控机制，再加一条更稳定的记忆通道。GRU 的答案是：保留核心思想，但把结构做得更轻一些。

这一阶段的重点，不是“模型更复杂了”，而是“信息流开始可控了”。

### 3. Seq2Seq：让输入序列和输出序列脱钩

当大家开始做机器翻译、摘要、对话时，会遇到一个新问题：输入和输出都不是一个定长向量，也不一定等长。

这时候 Seq2Seq（Sequence to Sequence）就出现了。它把任务拆成两半：

- 编码器先把输入序列读完
- 解码器再把输出序列一步步生成出来

这个思路很自然，也很符合人类直觉：先理解，再表达。

但 Seq2Seq 很快又撞上了一个瓶颈：编码器如果必须把整句内容都塞进一个固定长度向量里，那么长句信息会被严重压缩。

### 4. Attention：让模型别死记，学会“回头看”

Attention 的出现，本质是在说：

**既然固定长度向量装不下所有细节，那就别逼模型一次性全记住。生成每一步时，允许它回头去看输入序列里最相关的部分。**

这一步是关键转折。模型不再只依赖“压缩后的记忆”，而开始依赖“动态访问历史”。从这里再往前一步，就自然走向了 Transformer。

### 5. Transformer：把“回头看”变成主角

Transformer 并不是从天而降的全新物种。它可以被看作一个清晰的演化结果：既然 Attention 比单纯循环更擅长建模长距离关系，那就把 Attention 从辅助机制升级成主结构。

所以学 RNN 系列，不是为了回到旧时代，而是为了理解：**现代序列模型为什么会演化成今天这个样子。**

## 五、读完这一组后，你应该得到什么？

读完这个子模块，最重要的不是背下所有公式，而是建立下面这条因果链：

- 前馈网络不擅长处理序列
- RNN 通过隐状态引入记忆
- 原始 RNN 训练不稳，长依赖学不好
- LSTM / GRU 用门控改善信息流
- Seq2Seq 解决“序列到序列”的任务形态
- Attention 解决“固定长度记忆瓶颈”
- Transformer 则把 Attention 推到中心位置

如果这条线你看清了，后面再学 Transformer、大语言模型、语音模型、多模态模型时，会轻松很多。
