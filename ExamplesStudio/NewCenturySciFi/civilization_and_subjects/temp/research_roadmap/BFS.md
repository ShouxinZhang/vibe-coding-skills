# BFS

这不是研究计划书，而是一个可以直接开聊的问题池。

如果 DFS 是“主线怎么挖”，那 BFS 就是“有哪些点一开口就能讨论”。

## 先把边界聊明白

- 这里说的 OpenClaw，到底是某个具体产品，还是“人接入 agent 系统的入口”这个更大的概念？
- 我们现在谈的是内容平台、关系平台，还是更底层的数据主权和身份系统？
- 什么属于 2026 年附近还能落地的判断，什么已经是明显的远景脑洞？
- “平台会不会死”这个问题是不是问早了，应该先问“平台哪一层先被改写”？

## 内容平台那边，最值得聊什么

- 抖音这类平台最先被 agent 拆掉的，会是内容生产、内容分发，还是内容消费入口？
- “数据透明”到底能不能推导出“内容平台没必要存在”？
- 用户以后是真的不看平台了，还是平台变成了 agent 背后的素材库、流量池和广告市场？
- Stack Overflow 的变化，最多能类比到哪一步，不能类比到哪一步？
- 如果内容平台不死，只是职能变了，那它最像会变成什么？

## 关系平台这边，真正难的是什么

- 微信 / QQ 的核心到底是聊天界面，还是关系协议？
- agent 进来之后，是替代真人，还是只是变成真人的前台？
- 以后“加我微信”会不会慢慢变成“接入我的某个代理入口”？
- 好友关系如果进一步细化成权限关系，谁来决定 agent 能看什么、能做什么、能代表谁？
- 平台会主动开放关系图吗，还是只会做“平台内 agent 化”？

## 人和 agent 的关系，哪些点最有意思

- agent 是工具，还是人的分身，还是人的某一面被制度化之后的接口？
- 一个人如果有工作 agent、创作 agent、社交 agent、治理 agent，这还是“一个人”吗？
- 以后人与人交流，会不会越来越像“组织和组织之间的沟通”？
- 如果大部分社交都先经过 agent，真人本身还剩下什么不可替代的价值？
- 外貌、表达能力、知识密度这些传统社交优势，会不会被 agent 部分抹平？

## 关于“通信历史主权”，可以直接聊这些

- 用户真正想拿回的，到底是数据本身，还是自己和别人互动过的那段历史？
- 聊天记录为什么比普通文档更难说“完全属于我”？
- 平台为什么愿意给你备份、导出、迁移一部分，但不愿意给你完整 CRUD？
- 人和 agent 的对话记录，和人与人的聊天记录，应该被同样对待吗？
- 如果以后要换平台，最值钱的是账号、好友列表，还是那整段可继续工作的上下文？

## 关于分层和治理，可以直接打起来的问题

- agent 时代最关键的差距，还是钱吗？还是“谁拥有更强的系统配置权”？
- 真正拉开差距的是模型、数据、算力，还是谁拿得到更高级的接口和权限？
- 中国语境下，实名、备案、联网治理，会把 agent 社交压成什么样子？
- 联网 agent 最后会不会像今天的支付、云服务一样，进入强监管基础设施状态？
- “人人都有 agent”这句话，会不会像“人人都有互联网”一样，表面平等，实际分层更狠？

## 高假设脑洞，先别急着写进主线

- 机器人替身什么时候才算主线，不再只是支线脑洞？
- 意识跳跃、具身化 agent、元宇宙式多时空存在，应该放进哪一篇？
- 如果这些高假设成立，最先被改写的是社交、劳动，还是人的存在方式本身？
- 什么条件没成立之前，这些内容都应该老实待在附录里？

## 一些容易混淆的词

- `数据透明` 不等于 `平台消亡`
- `社交数据` 不等于 `普通私有文件`
- `关系平台` 不等于 `聊天窗口`
- `数据主权` 不等于 `通信历史主权`
- `agent 帮忙做事` 不等于 `agent 已经成为社会主体`

## 一些值得单独开篇的小命题

- 内容平台先被工作流化，关系平台后被 agent 化。
- 用户真正想拿回的不是全部数据，而是自己的通信历史连续性。
- agent 社交不是消灭社交平台，而是改写社交平台的接入层。
- 未来人与人之间的沟通，越来越像“人带着自己的代理组织去沟通”。
- agent 时代拉开差距的，不只是财富，而是高质量系统配置权。

## 如果只想先挑一个点继续写

优先从这三个问题里挑：

1. 为什么内容平台和关系平台不能放在同一套逻辑里讨论？
2. 为什么“通信历史主权”比“数据主权”更贴近你真正想说的东西？
3. 为什么 agent 社交的核心不是替人回消息，而是把“人”改写成多层代理组织？
