# 边界与变量

## 这一块在讨论什么

先搞清楚我们到底在讨论什么层级。

这里的重点不是某个平台的细节，而是把几个总是混在一起的东西拆开：

- 内容平台
- 关系平台
- 数据主权
- 通信历史主权
- agent 作为工具
- agent 作为社会接口

如果这一步不先立住，后面所有判断都会互相串台。

## 为什么值得单独拆出来

原始草稿的最大问题之一，就是概念跳得太快。

前面一句还在说数据透明，后面一句就跳到平台消亡；上一段还在谈内容创作，下一段就开始谈微信/QQ 的终局。读的时候会有一种“每句话都对，但不是在同一个层级上对”的感觉。

所以第一块必须先做边界清理。

## 关键问题

- OpenClaw 在这里到底是具体产品，还是“人与 agent 系统的入口”这个抽象角色？
- 内容平台和关系平台的边界到底怎么划？
- 什么属于用户侧工作区，什么属于平台关系资产？
- 哪些判断是近中期趋势，哪些已经是高假设远景？
- 后面每个分支要共用哪一套术语？

## 当前直觉

最稳的做法是先立四层：

1. 内容生产与消费层
2. 社会关系与身份层
3. 数据存储与权属层
4. 治理与合规层

把这四层分开之后，你会发现很多原本看起来在争同一个问题的句子，其实根本不是在回答同一件事。

## 后续怎么扩写

1. 补一张术语对照表，把每个高频词都钉死。
2. 给每一层配上典型平台例子，避免后面继续抽象打架。
3. 额外加一节“哪些混淆最常见”，专门拦截后续写作时会反复犯的错误。
