# 关系平台为什么更像协议层

## 这一块在讨论什么

这一块只讨论微信 / QQ 这类关系平台。

核心问题是：它们到底是在提供一个聊天界面，还是在提供一套更深的社会连接协议？

## 为什么值得单独拆出来

内容平台和关系平台不能混着讲。

内容平台更像内容供给与分发系统，关系平台更像身份入口、权限系统、服务分发层和连接协议。二者的粘性来源不同，被 agent 改写的顺序和方式也不同。

如果不把这一块单独拉出来，后面就会不断出现一种假象：好像抖音怎么变，微信就会怎么变。实际上完全不是一回事。

## 关键问题

- 微信 / QQ 的真正护城河到底是什么？
- 为什么它们不只是“消息 UI”，而更像“社会连接协议 + 身份入口 + 权限系统 + 服务分发层”？
- agent 进来之后，最先改变的是账号、权限、关系图，还是服务接口？
- 平台更可能走“平台内 agent 化”，还是“主动开放关系图”？

## 当前直觉

关系平台最难被替代的，不是聊天窗口，而是“谁能稳定触达到谁”这件事。

所以更现实的演化路径，不是微信/QQ 突然消失，而是：

1. agent 先变成默认前台
2. 账号变成 root account
3. 好友关系进一步细化成权限关系
4. 服务号、小程序、群聊逐步 agent 化

也就是说，关系平台更可能被 agent 改写使用方式，而不是立刻失去基础设施地位。
