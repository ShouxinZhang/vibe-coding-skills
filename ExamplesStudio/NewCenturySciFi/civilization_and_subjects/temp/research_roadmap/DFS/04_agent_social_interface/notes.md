# agent 的社交接口

## 这一块在讨论什么

这一块讨论的不是平台，而是人。

准确地说，是讨论：当一个人开始拥有多个 agent，而且这些 agent 分别代表工作的自己、创作的自己、社交的自己时，人会怎样从一个单点账号，慢慢变成一个多层代理组织。

## 为什么值得单独拆出来

原始草稿最有价值的一个判断，其实不在平台，而在这里：

agent 的社交意义，不只是替人做事，而是把“人”重新组织成了一个带多层代理的微型组织。

这件事如果讲清楚，后面很多问题都会顺下来。比如为什么“加我微信”会慢慢变成“接入我的某个代理入口”，为什么人与人之间的沟通会越来越像组织与组织之间的沟通。

## 关键问题

- agent 到底是工具、分身，还是人的某个社会面向的外包接口？
- 一个人拥有多个 agent 之后，哪些权限可以放给 agent，哪些必须保留给真人？
- agent 之间如何分工、如何共享记忆、如何接受最终授权？
- 当外界接触到的是你的 agent，而不是你本人时，“你是谁”这个问题会怎么变？

## 当前直觉

未来更像是：

- 真人是最终主权者
- 多个 agent 分别承担不同前台
- 外界先接入代理层，再决定要不要升级到真人层

在这种结构里，人不是消失了，而是被重新组织了。
