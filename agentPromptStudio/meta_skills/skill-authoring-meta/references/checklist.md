# 验收清单

## 结构

- [ ] 目录是否为 `SKILL.md + references/ + scripts/`。
- [ ] `references/` 是否包含模块化 `md` 文档（>=2）。
- [ ] `scripts/` 是否仅包含 `*.sh` 文件。
- [ ] `SKILL.md` 是否给出可点击的子模块入口路径。

## 脚本

- [ ] 脚本是否具备执行权限（`chmod +x`）。
- [ ] 至少运行一个脚本验证无语法错误。
- [ ] `SCRIPT_DIR` → repo root 的 `..` 层级是否正确（标准为4层）。
- [ ] 启动后台进程的脚本是否有 `trap cleanup EXIT`。
- [ ] 外部依赖是否用 `npx -y` 或在文档中声明。

## SKILL.md

- [ ] `description` 是否包含 "Use when:" 触发词。
- [ ] "默认约束"中是否注明产出物存放路径。
- [ ] 脚本依赖的外部工具是否有声明。

## 闭环测试

- [ ] 实际执行一遍完整流程，确认产出物生成。
- [ ] 产出物不污染源码目录（存放于 `output/` 子目录）。
