# 案例入口

开源站点真正能打动外部访客的，不只是理念，而是“这个仓库能产出什么”。

## 当前最适合展示的案例类型

### 1. 可运行小应用

[`ExamplesStudio/ModularSeparation/Sokoban`](https://github.com/ShouxinZhang/vibe-coding-skills/tree/main/ExamplesStudio/ModularSeparation/Sokoban) 是最直接的展示素材。它适合作为：

- 前端示例入口
- 模块化设计示意
- 从知识到实践的桥接证明

### 2. 方法论内容页

`BasicKnowledge` 中的内容适合拆成专题展示，例如：

- Agent Workflow
- Meta Code
- Agent System

这些主题的优势是结构清晰、阅读连续性强、适合被搜索引擎索引。

### 3. 仓库导览型页面

对于第一次接触项目的人，比起一上来阅读深文章，更需要一个总览页面告诉他们：

- 这个项目是什么
- 能解决什么问题
- 先看什么最划算

这个站点就是为这个目的而建。

## 第二阶段可以补什么

- 给 `Sokoban` 单独做在线体验跳转
- 给高价值专题做精选页，而不是把所有文章原样搬运
- 增加“推荐阅读路径”和“适用场景”标签，提升内容消费效率
